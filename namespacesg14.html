specializes fixed_point with the given number of integer and fractional digits

Template Parameters
IntegerDigitsspecifies minimum value of fixed_point::integer_digits
FractionalDigitsspecifies the exact value of fixed_point::fractional_digits
Archetypehints at the type of fixed_point::rep
Remarks
The signage of Archetype specifies signage of the resultant fixed-point type.
Typical choices for Archetype, signed and unsigned, result in a type that uses built-in integers for fixed_point::rep.
Resultant type is signed by default.
Example:

To generate a fixed-point type with a sign bit, 8 fractional bits and at least 7 integer bits:

static_assert(std::is_same<make_fixed<7, 8, signed>, fixed_point<int16_t, -8>>::value, "failed to use make_fixed");

template<std::int64_t Value, class Archetype = int>
constexpr auto sg14::elasticate ( ) -> _elastic_impl::elasticate_t<std::int64_t, Value, Archetype>

generate an Extensible object of given value

Template Parameters
Valuethe integer number to be represented
Archetypethe archetype of the resultant Extensible object
Returns
the given value represented using an Extensible type
Note
The return type is guaranteed to be no larger than is necessary to represent the value.
Example

To define a 1-byte object with value 1024:

constexpr auto n = elasticate<1024, char>();
static_assert(n==1024, "n now has the value, 1024");
static_assert(sizeof(n)==sizeof(char), "by default elasticate uses the most eficient type it can");

To define a int-sized object with value 1024:

constexpr auto n = elasticate<1024>();
static_assert(n==1024, "n now has the value, 1024");
static_assert(sizeof(n)==sizeof(int), "by default elasticate uses the most eficient type it can");
